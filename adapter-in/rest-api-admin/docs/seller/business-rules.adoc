[[seller-business-rules]]
= 셀러 비즈니스 규칙

이 문서는 셀러 관리 시스템의 핵심 비즈니스 규칙을 정의합니다.

== 입점 신청 규칙

=== 신청 제출 규칙

[cols="1,3,1"]
|===
| 규칙 ID | 설명 | 검증 시점

| APP-001
| 셀러명은 필수이며 빈 값일 수 없습니다
| 신청 시

| APP-002
| 사업자등록번호는 10자리 숫자 형식이어야 합니다
| 신청 시

| APP-003
| 반품 주소 정보는 필수입니다
| 신청 시

| APP-004
| CS 연락처 중 최소 하나는 필수입니다 (전화번호 또는 이메일)
| 신청 시
|===

=== 신청 상태 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| APP-010
| 신청은 `PENDING` 상태에서만 승인/거절할 수 있습니다

| APP-011
| 이미 `APPROVED` 또는 `REJECTED` 상태인 신청은 상태를 변경할 수 없습니다

| APP-012
| 거절 시 거절 사유는 필수입니다
|===

== 승인 처리 규칙

=== 중복 검사 규칙

[cols="1,3,2"]
|===
| 규칙 ID | 설명 | 에러 코드

| APR-001
| 셀러명은 시스템 내에서 고유해야 합니다 (중복 불가)
| `SELLER_NAME_DUPLICATE`

| APR-002
| 사업자등록번호는 시스템 내에서 고유해야 합니다 (중복 불가)
| `REGISTRATION_NUMBER_DUPLICATE`
|===

=== Seller 생성 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| APR-010
| 승인 시 Seller, SellerAddress, SellerBusinessInfo, SellerCs가 함께 생성됩니다

| APR-011
| 생성된 Seller는 기본적으로 `active=true` 상태입니다

| APR-012
| 신청에 입력된 주소는 `RETURN` 타입의 기본 주소로 생성됩니다
|===

== 셀러 관리 규칙

=== 셀러 정보 수정 규칙

[cols="1,3,2"]
|===
| 규칙 ID | 설명 | 적용 대상

| SEL-001
| 셀러명 변경 시 중복 검사를 수행합니다 (자기 자신 제외)
| 기본 정보 수정

| SEL-002
| 비활성화된 셀러는 정보 수정이 제한됩니다
| 모든 수정 API

| SEL-003
| 삭제된 셀러는 조회 및 수정할 수 없습니다
| 모든 API
|===

=== 셀러 상태 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| SEL-010
| 이미 활성화된 셀러를 다시 활성화할 수 없습니다

| SEL-011
| 이미 비활성화된 셀러를 다시 비활성화할 수 없습니다

| SEL-012
| 삭제는 소프트 삭제(Soft Delete)로 처리됩니다
|===

== 사업자 정보 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| BIZ-001
| 사업자등록번호 변경 시 중복 검사를 수행합니다

| BIZ-002
| 통신판매업 신고번호는 선택 사항입니다

| BIZ-003
| 사업장 주소는 사업자등록증 상의 주소와 일치해야 합니다
|===

== 주소 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| ADR-001
| 셀러는 최소 하나의 반품 주소(RETURN)를 가져야 합니다

| ADR-002
| 각 주소 타입별로 하나의 기본 주소만 설정할 수 있습니다

| ADR-003
| 우편번호는 5자리 숫자 형식이어야 합니다
|===

== CS 정보 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| CS-001
| 운영 시간은 HH:mm 형식이어야 합니다 (예: 09:00)

| CS-002
| 운영 종료 시간은 시작 시간보다 이후여야 합니다

| CS-003
| 운영 요일은 쉼표로 구분된 요일 코드입니다 (MON,TUE,WED,THU,FRI,SAT,SUN)
|===

== 계약 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| CON-001
| 수수료율은 0% 이상 100% 이하여야 합니다

| CON-002
| 계약 종료일은 시작일 이후여야 합니다

| CON-003
| 계약 상태 전이: PENDING → ACTIVE → EXPIRED 또는 TERMINATED
|===

== 정산 규칙

[cols="1,3"]
|===
| 규칙 ID | 설명

| SET-001
| 정산 계좌는 계좌 인증 후에만 정산 처리에 사용됩니다

| SET-002
| 정산일은 1일부터 31일 사이의 값이어야 합니다

| SET-003
| 정산 주기는 WEEKLY, BIWEEKLY, MONTHLY 중 하나입니다
|===

== 에러 코드 정의

=== 입점 신청 관련 에러

[cols="2,3,1"]
|===
| 에러 코드 | 설명 | HTTP 상태

| `SELLER_APPLICATION_NOT_FOUND`
| 입점 신청을 찾을 수 없습니다
| 404

| `SELLER_APPLICATION_ALREADY_PROCESSED`
| 이미 처리된 입점 신청입니다
| 400

| `REJECT_REASON_REQUIRED`
| 거절 사유는 필수입니다
| 400
|===

=== 셀러 관련 에러

[cols="2,3,1"]
|===
| 에러 코드 | 설명 | HTTP 상태

| `SELLER_NOT_FOUND`
| 셀러를 찾을 수 없습니다
| 404

| `SELLER_NAME_DUPLICATE`
| 이미 사용 중인 셀러명입니다
| 400

| `SELLER_ALREADY_ACTIVE`
| 이미 활성화된 셀러입니다
| 400

| `SELLER_ALREADY_INACTIVE`
| 이미 비활성화된 셀러입니다
| 400

| `SELLER_INACTIVE`
| 비활성화된 셀러입니다
| 400
|===

=== 사업자 정보 관련 에러

[cols="2,3,1"]
|===
| 에러 코드 | 설명 | HTTP 상태

| `REGISTRATION_NUMBER_DUPLICATE`
| 이미 등록된 사업자등록번호입니다
| 400

| `BUSINESS_INFO_NOT_FOUND`
| 사업자 정보를 찾을 수 없습니다
| 404
|===
