[[policy-rules]]
= 정책 비즈니스 규칙

배송 정책(ShippingPolicy)과 환불 정책(RefundPolicy)에 적용되는 비즈니스 규칙을 정의합니다.

== 기본 정책 (Default Policy) 규칙

=== 불변 조건

[cols="1,3,2"]
|===
| 규칙 ID | 규칙 | 설명

| POL-DEF-001
| 셀러당 기본 정책은 **정확히 1개**
| 0개도 안 되고, 2개 이상도 안 됨

| POL-DEF-002
| 기본 정책은 **활성화 상태**여야 함
| 비활성화된 정책은 기본 불가

| POL-DEF-003
| 삭제된 정책은 기본이 될 수 없음
| soft delete 시에도 적용
|===

=== 기본 정책 변경 흐름

[source]
----
[Before]
- 정책 A (defaultPolicy = true)  ← 기본
- 정책 B (defaultPolicy = false)

[B를 기본으로 변경 요청]
         │
         ▼
┌─────────────────────────────────┐
│ 1. B 조회 (새 기본 정책)          │
│ 2. B가 활성화 상태인지 검증        │
│ 3. 기존 기본 정책 A 조회           │
│ 4. A.unmarkDefault()            │
│ 5. B.markAsDefault()            │
│ 6. 둘 다 persist (같은 트랜잭션)   │
└─────────────────────────────────┘
         │
         ▼
[After]
- 정책 A (defaultPolicy = false)
- 정책 B (defaultPolicy = true)  ← 기본
----

== 정책 등록 규칙

=== 첫 번째 정책 등록

[cols="1,3,2"]
|===
| 규칙 ID | 규칙 | 설명

| POL-REG-001
| 셀러의 첫 정책은 **자동으로 기본 정책**
| `defaultPolicy` 요청값과 무관하게 true

| POL-REG-002
| 첫 정책은 **자동으로 활성화**
| `active = true`
|===

=== 기본 정책으로 등록 요청 시

[cols="1,3,2"]
|===
| 규칙 ID | 규칙 | 설명

| POL-REG-003
| `defaultPolicy=true`로 등록 시 기존 기본 해제
| 기존 기본 정책의 `defaultPolicy`를 false로 변경
|===

[source]
----
[현재 상태]
- 정책 A (기본)

[새 정책 B 등록, defaultPolicy = true]
         │
         ▼
┌─────────────────────────────────┐
│ 1. 기존 기본 정책 A 조회           │
│ 2. A.unmarkDefault()            │
│ 3. B 생성 (defaultPolicy = true) │
│ 4. 둘 다 persist                 │
└─────────────────────────────────┘
         │
         ▼
[결과]
- 정책 A (defaultPolicy = false)
- 정책 B (defaultPolicy = true)  ← 기본
----

== 정책 삭제 규칙

=== 삭제 제약 조건

[cols="1,3,2"]
|===
| 규칙 ID | 규칙 | 에러 코드

| POL-DEL-001
| 기본 정책은 삭제 불가
| `CANNOT_DELETE_DEFAULT_POLICY`

| POL-DEL-002
| 마지막 활성 정책은 삭제 불가
| `LAST_ACTIVE_POLICY_CANNOT_BE_DELETED`

| POL-DEL-003
| 상품에 연결된 정책은 삭제 불가 (선택)
| `POLICY_IN_USE`
|===

=== 삭제 흐름

[source]
----
[삭제 요청]
     │
     ▼
┌─────────────────────────────────┐
│ 1. 기본 정책인가?                 │
│    → YES: 에러 "먼저 다른 정책을   │
│           기본으로 지정해주세요"    │
│    → NO: 계속                    │
├─────────────────────────────────┤
│ 2. 마지막 활성 정책인가?           │
│    → YES: 에러 "최소 1개 정책 필요" │
│    → NO: 계속                    │
├─────────────────────────────────┤
│ 3. 사용 중인 상품이 있는가? (선택)  │
│    → YES: 에러 "사용 중인 정책"    │
│    → NO: 계속                    │
├─────────────────────────────────┤
│ 4. soft delete 실행              │
│    deletedAt = now              │
└─────────────────────────────────┘
----

== 정책 수정 규칙

=== 수정 가능 항목

[cols="2,1,3"]
|===
| 항목 | 수정 가능 | 비고

| 정책명
| O
|

| O

== 배송 정책 (ShippingPolicy) 에러 코드

[cols="2,1,3"]
|===
| 에러 코드 | HTTP 상태 | 메시지

| `SHIPPING_POLICY_NOT_FOUND`
| 404
| 배송 정책을 찾을 수 없습니다

| `CANNOT_DELETE_DEFAULT_POLICY`
| 400
| 기본 배송 정책은 삭제할 수 없습니다. 먼저 다른 정책을 기본으로 지정해주세요

| `CANNOT_DEACTIVATE_DEFAULT_POLICY`
| 400
| 기본 배송 정책은 비활성화할 수 없습니다. 먼저 다른 정책을 기본으로 지정해주세요

| `LAST_ACTIVE_POLICY_CANNOT_BE_DELETED`
| 400
| 최소 1개의 활성 배송 정책이 필요합니다

| `LAST_ACTIVE_POLICY_CANNOT_BE_DEACTIVATED`
| 400
| 최소 1개의 활성 배송 정책이 필요합니다

| `INACTIVE_POLICY_CANNOT_BE_DEFAULT`
| 400
| 비활성화된 정책은 기본으로 지정할 수 없습니다

| `INVALID_FREE_THRESHOLD`
| 400
| 조건부 무료배송은 무료배송 기준금액이 필요합니다
|===

| `CANNOT_DEACTIVATE_DEFAULT_POLICY`
| 400
| 기본 환불 정책은 비활성화할 수 없습니다. 먼저 다른 정책을 기본으로 지정해주세요

| `LAST_ACTIVE_POLICY_CANNOT_BE_DELETED`
| 400
| 최소 1개의 활성 환불 정책이 필요합니다

| `LAST_ACTIVE_POLICY_CANNOT_BE_DEACTIVATED`
| 400
| 최소 1개의 활성 환불 정책이 필요합니다

| `INACTIVE_POLICY_CANNOT_BE_DEFAULT`
| 400
| 비활성화된 정책은 기본으로 지정할 수 없습니다

| `INVALID_RETURN_PERIOD`
| 400
| 반품 기간은 1~90일 이내여야 합니다

| `INVALID_EXCHANGE_PERIOD`
| 400
| 교환 기간은 1~90일 이내여야 합니다
|===
