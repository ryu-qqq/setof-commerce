### QueryDSL 코드 (Recursive CTE)

```java
// CategoryFetchRepositoryImpl.fetchAllParentCategories()
public List<TreeCategoryContext> fetchAllParentCategories(long categoryId) {
    JPASQLQuery<Category> q = new JPASQLQuery<>(em, SQLTemplates.DEFAULT);
    QCategory c = new QCategory("c");
    QCategory sub = new QCategory("sub");
    EntityPathBase<QCategory> rec = new EntityPathBase<>(QCategory.class, "sub");

    // Base case: 기준 카테고리
    JPQLQuery<Category> query1 = JPAExpressions
        .select(Projections.fields(Category.class,
            c.id, c.categoryName, c.displayName, c.parentCategoryId, c.categoryDepth))
        .from(c)
        .where(c.id.eq(categoryId));

    // Recursive case: c.id = sub.parentCategoryId 인 상위 카테고리
    JPQLQuery<Category> query2 = JPAExpressions
        .select(Projections.fields(Category.class,
            c.id, c.categoryName, c.displayName, c.parentCategoryId, c.categoryDepth))
        .from(rec)
        .innerJoin(c).on(c.id.eq(sub.parentCategoryId));  // 🔑 상위 방향 조인

    Union<Category> union = SQLExpressions.unionAll(query1, query2);

    return q.withRecursive(rec, c.id, c.categoryName, c.displayName, c.parentCategoryId, c.categoryDepth)
        .as(union)
        .select(new QTreeCategoryContext(
            sub.id, sub.categoryName, sub.displayName, sub.categoryDepth, sub.parentCategoryId))
        .from(rec)
        .orderBy(sub.categoryDepth.desc())  // depth 내림차순
        .fetch();
}
```

### 조인 조건 비교 (하위 vs 상위)

| API | 조인 조건 | 탐색 방향 |
|-----|-----------|----------|
| fetchAllChildCategories | `sub.id = c.parentCategoryId` | 하위 방향 (자식 탐색) |
| fetchAllParentCategories | `c.id = sub.parentCategoryId` | **상위 방향 (부모 탐색)** |

```sql
-- 상위 카테고리 탐색
INNER JOIN category c ON c.id = sub.parentCategoryId

-- 의미: 부모 카테고리(c)의 ID가 현재 노드(sub)의 parentCategoryId와 같은 것
-- 결과: 기준 카테고리 → 부모 → 조부모 → ... → 루트 순으로 탐색
```
