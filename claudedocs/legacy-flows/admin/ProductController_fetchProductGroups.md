### 2-Phase Query 전략

**Phase 1: ID 목록 조회**
```java
getQueryFactory()
    .select(productGroup.id)
    .from(productGroup)
    .where(
        betweenTime(filter),
        managementTypeEq(filter),
        brandEq(filter),
        sellerIdEq(filter.getSellerId()),
        soldOutEq(filter),
        displayEq(filter),
        categoryIn(filter),
        betweenPrice(filter),
        betweenSalePercent(filter),
        searchKeywordEq(filter.getSearchKeyword(), filter.getSearchWord()),
        deleteYn()
    )
    .orderBy(ORDERS)
    .offset(pageable.getOffset())
    .limit(pageable.getPageSize())
    .fetch();
```

**Phase 2: 상세 조회 (ID IN)**
```java
getQueryFactory()
    .from(productGroup)
    .innerJoin(brand).on(brand.id.eq(productGroup.productGroupDetails.brandId))
    .innerJoin(category).on(category.id.eq(productGroup.productGroupDetails.categoryId))
    .innerJoin(seller).on(seller.id.eq(productGroup.productGroupDetails.sellerId))
    .innerJoin(productGroup.productDelivery, productDelivery)
    .innerJoin(productGroup.images, productGroupImage)
        .on(productGroupImage.deleteYn.eq(Yn.N))

    .leftJoin(externalProduct).on(externalProduct.productGroupId.eq(productGroup.id))
    .distinct()

    .orderBy(ORDERS)
    .transform(GroupBy.groupBy(productGroup.id).list(...))
```

## 📝 특이사항

1. **2-Phase Query 패턴**:
   - Phase 1: WHERE 조건으로 ID만 조회 (가벼운 쿼리)
   - Phase 2: ID IN으로 상세 데이터 조회 (JOIN 최적화)

2. **권한 기반 필터링**:
   - `@AuthorityValidate` AOP로 sellerId 자동 세팅
   - MASTER: 전체 조회, SELLER: 자신의 상품만

3. **No-Offset 페이징 지원**:
   - `lastDomainId` 파라미터로 커서 기반 페이징
   - 대용량 데이터에서 성능 향상

4. **카테고리 하위 조회**:
   - `@CategoryValidate` AOP로 하위 카테고리 자동 조회
   - categoryId → categoryIds (Set) 확장

5. **추가 조회**:
   - `CategoryFetchService.fetchAllCategories()`: 전체 경로명 조회
   - `ProductFetchService.fetchProducts()`: 개별 상품 조회
