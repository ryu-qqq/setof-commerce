    .from(qna)

### 조건 메서드 구현

```java
private BooleanExpression qnaStatusEq(QnaFilter filter) {
    if (filter.getQnaStatus() != null)
        return qna.qnaStatus.eq(filter.getQnaStatus());
    return null;
}

private BooleanExpression qnaTypeEq(QnaFilter filter) {
    if (filter.getQnaType() != null)
        return qna.qnaType.eq(filter.getQnaType());
    return null;
}

private BooleanExpression betweenTime(QnaFilter filter) {
    return qna.insertDate.between(filter.getStartDate(), filter.getEndDate());
}

private BooleanExpression isCursorRead(QnaFilter dto) {
    if (dto.getLastDomainId() != null && dto.getLastDomainId() > 0) {
        return qna.id.lt(dto.getLastDomainId());
    }
    return null;
}
```

## 📋 특이사항

### 1. 2-Phase Query 패턴
- **Phase 1**: ID 목록만 조회 (페이징 + 필터 적용)
- **Phase 2**: ID IN 절로 상세 데이터 조회
- 목적: N+1 문제 방지 + 페이징 최적화

### 2. 다형성 분기 처리
- `filter.isProductQna()` → 상품 문의용 쿼리
- `else` → 주문 문의용 쿼리 (스냅샷 테이블 조인)

### 3. 권한 처리
- `@AuthorityValidate` 어노테이션으로 sellerId 자동 주입
- `RoleFilter` 인터페이스 구현

### 4. 정렬
- 기본 정렬: `qna.id.desc()` (최신순)

### 5. 페이징 방식
- Offset 페이징 + Cursor 페이징 모두 지원
- `lastDomainId` 있으면 커서 페이징 적용
