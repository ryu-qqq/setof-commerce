```java
getQueryFactory()
```

### 2-A. 상품 문의 조회 (fetchProductQna)

#### 조회 테이블

| 테이블 | Alias | 조인 유형 | 조인 조건 |
|--------|-------|----------|----------|
| qna | qna | FROM | - |
| qna_answer | qnaAnswer | LEFT JOIN | qnaAnswer.qna.id = qna.id |
| qna_product | qnaProduct | INNER JOIN | qna.qnaProduct (OneToOne) |
| product_group | productGroup | INNER JOIN | productGroup.id = qnaProduct.productGroupId |
| brand | brand | INNER JOIN | brand.id = productGroup.brandId |
| product_group_image | productGroupImage | INNER JOIN | images 컬렉션 + MAIN 타입 + deleteYn=N |
| seller | seller | INNER JOIN | seller.id = productGroup.sellerId |
| users | users | INNER JOIN | users.id = qna.userId |

#### QueryDSL 코드

```java
getQueryFactory()
    .from(qna)
    .leftJoin(qnaAnswer).on(qnaAnswer.qna.id.eq(qna.id))
    .innerJoin(qna.qnaProduct, qnaProduct)
    .innerJoin(productGroup).on(productGroup.id.eq(qnaProduct.productGroupId))
    .innerJoin(brand).on(brand.id.eq(productGroup.productGroupDetails.brandId))
    .innerJoin(productGroup.images, productGroupImage)
        .on(productGroupImage.imageDetail.productGroupImageType.eq(ProductGroupImageType.MAIN))
        .on(productGroupImage.deleteYn.eq(Yn.N))
    .innerJoin(seller).on(seller.id.eq(productGroup.productGroupDetails.sellerId))
    .innerJoin(users).on(users.id.eq(qna.userId))
    .where(qnaIdEq(qnaId), sellerIdEq(sellerIdOpt))
    .transform(GroupBy.groupBy(qna.id).as(...))
```

### 2-B. 주문 문의 조회 (fetchOrderQna)

#### 조회 테이블

| 테이블 | Alias | 조인 유형 | 조인 조건 |
|--------|-------|----------|----------|
| qna | qna | FROM | - |
| qna_answer | qnaAnswer | LEFT JOIN | qnaAnswer.qna.id = qna.id |
| qna_order | qnaOrder | INNER JOIN | qna.qnaOrder (OneToOne) |
| qna_image | qnaImage | LEFT JOIN | qna.qnaImages + deleteYn=N |

| orders | order | INNER JOIN | order.id = qnaOrder.orderId |
| order_snapshot_product_group | orderSnapShotProductGroup | INNER JOIN | order.orderSnapShotProductGroup |
| brand | brand | INNER JOIN | brand.id = snapshot.brandId |
| seller | seller | INNER JOIN | seller.id = qna.sellerId |
| users | users | INNER JOIN | users.id = qna.userId |
| order_snapshot_product_group_image | orderSnapShotProductGroupImage | JOIN | MAIN 이미지 |
| order_snapshot_option_detail | orderSnapShotOptionDetail | LEFT JOIN | 옵션 상세 |

#### QueryDSL 코드

```java
QQnaImage qnaImageAnswer = new QQnaImage("qnaImageAnswer");

getQueryFactory()
    .from(qna)
    .leftJoin(qnaAnswer).on(qnaAnswer.qna.id.eq(qna.id))
    .innerJoin(qna.qnaOrder, qnaOrder)
    .leftJoin(qna.qnaImages, qnaImage).on(qnaImage.deleteYn.eq(Yn.N))
    .leftJoin(qnaAnswer.qnaImages, qnaImageAnswer).on(qnaImageAnswer.deleteYn.eq(Yn.N))
    .innerJoin(order).on(order.id.eq(qnaOrder.orderId))
    .innerJoin(order.orderSnapShotProductGroup, orderSnapShotProductGroup)
    .innerJoin(brand).on(brand.id.eq(orderSnapShotProductGroup.snapShotProductGroup.brandId))
    .innerJoin(seller).on(seller.id.eq(qna.sellerId))
    .innerJoin(users).on(users.id.eq(qna.userId))
    .join(order.orderSnapShotProductGroupImages, orderSnapShotProductGroupImage)
        .on(imageType.eq(ProductGroupImageType.MAIN))
    .leftJoin(order.orderSnapShotOptionDetails, orderSnapShotOptionDetail)
    .where(qnaIdEq(qnaId), sellerIdEq(sellerIdOpt))
    .transform(GroupBy.groupBy(qna.id).as(...))
```

## 📋 특이사항

### 1. 다형성 처리
- `QnaType`에 따라 다른 쿼리 실행 (상품문의 vs 주문문의)
- `QnaTarget` 인터페이스로 응답 다형성 처리

### 2. GroupBy Transform
- QueryDSL `GroupBy.groupBy().as()` 패턴 사용
- 1:N 관계를 Set/List로 그룹핑

### 3. 권한 처리
- `SecurityUtils.currentSellerIdOpt()` → 셀러 권한이면 해당 셀러의 QnA만 조회
- 마스터 권한이면 전체 조회 가능

### 4. 이미지 후처리 (주문문의)
- `DetailQnaResponse.setQnaImages()` 에서 QnA/답변 이미지 매핑
