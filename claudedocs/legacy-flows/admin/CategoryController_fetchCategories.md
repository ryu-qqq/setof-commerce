### WHERE 조건

| 조건 | 메서드 | 로직 |
|------|--------|------|

| categoryIdEq | `id.eq(categoryId)` | categoryId가 null이면 조건 없음 |
| categoryDepthEq | `categoryDepth.eq(depth)` | categoryDepth가 null이면 조건 없음 |
| getCategoryIdCondition | ASC: `id.gt(lastId)` / DESC: `id.lt(lastId)` | No-Offset 페이징 조건 |
| exclusiveNonDisplayCategory | `id.notIn(1828)` | 비노출 카테고리 제외 |

## 📝 마이그레이션 노트

### 특이사항

1. **No-Offset 페이징 지원**
   - `lastDomainId`가 있으면 No-Offset 방식 사용
   - 대용량 데이터에서 성능 이점
   - 새 아키텍처에서도 동일하게 지원 필요

2. **검색 필드 불일치**
   - Filter의 `categoryName`이 실제로는 `displayName`으로 검색됨
   - 새 아키텍처에서 필드명 일관성 개선 고려

3. **카운트 쿼리 조건 불일치**
   - 데이터 쿼리: `categoryIdEq`, `categoryDepthEq` 포함
   - 카운트 쿼리: `exclusiveNonDisplayCategory`만 포함
   - 필터 조건이 카운트에 반영되지 않는 버그 가능성
