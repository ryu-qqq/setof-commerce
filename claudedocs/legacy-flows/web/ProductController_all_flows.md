# 📝 특이사항

1. **쿼리 최적화**: `fetchProductGroupDto`는 3개의 분리된 쿼리로 N+1 문제 방지
2. **커서 기반 페이징**: 대용량 데이터 처리를 위한 커서 페이징 지원
3. **동적 정렬**: 8가지 OrderType 지원 (최신순, 가격순, 할인율순, 평점순, 리뷰순, 추천순)
4. **Redis 캐시**: 브랜드별/셀러별 상품 목록은 Redis 캐시 적용
5. **GroupBy Transform**: QueryDSL GroupBy를 활용한 효율적인 1:N 관계 조회
6. **공통 추상 클래스**: 상품 목록 조회 로직 재사용을 위한 AbstractComponentRepository
