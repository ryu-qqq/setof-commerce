## 📊 fetchProductGroupReviews vs fetchMyReviews 비교

| **페이징** | offset 기반 (Page) | 커서 기반 (CustomSlice) |

## 📊 성능 고려사항

1. **2단계 쿼리 패턴**: ID 조회 → 상세 조회 (N+1 방지)
2. **커서 기반 페이징**: offset 없이 lastDomainId로 효율적 페이징
3. **GroupBy Transform**: 1:N 관계 데이터를 메모리에서 그룹핑
4. **삭제 리뷰 제외**: deleteYn = 'N' 조건으로 소프트 삭제 지원
