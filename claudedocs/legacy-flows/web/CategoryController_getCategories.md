## 🌳 트리 구조 변환 로직

```java
// CategoryFindServiceImpl.constructTree()
private List<CategoryDisplayDto> constructTree(List<CategoryDisplayDto> categories) {
    // 1. categoryId → DTO 맵 생성
    Map<Long, CategoryDisplayDto> categoryMap = categories.stream()
            .collect(Collectors.toMap(
                    CategoryDisplayDto::getCategoryId,
                    Function.identity()));

    // 2. depth=1은 루트, 그 외는 부모의 children에 추가
    List<CategoryDisplayDto> roots = new ArrayList<>();
    for (CategoryDisplayDto category : categories) {
        CategoryDisplayDto dto = categoryMap.get(category.getCategoryId());
        if (category.getCategoryDepth() == 1) {
            roots.add(dto);
        } else {
            CategoryDisplayDto parent = categoryMap.get(category.getParentCategoryId());
            parent.getChildren().add(dto);
        }
    }
    return roots;
}
```

## ⚠️ 주의사항

1. **하드코딩된 ID 제외**: `category.id.notIn(1828L)` - 특정 카테고리 ID가 하드코딩되어 있음
2. **displayName vs categoryName**: QueryDSL에서 `displayName`을 사용하지만 DTO 필드명은 `categoryName`
3. **Redis 캐시 의존**: 캐시가 없으면 매번 전체 카테고리 조회 + 트리 변환 수행
4. **트리 변환 로직**: depth=1이 루트, 나머지는 parentCategoryId로 부모 찾아서 children에 추가
